# AIBrainIM P1 可用版闭环说明

## 本轮目标

以 React Native 主工程为唯一主线，先打通一个用户真的能看、能走、能理解的 AI 大脑移动端闭环，而不是继续堆 HTML 体验稿或纯开发者信息。

## 当前已完成

### 1. 五主功能骨架固定
- 总览
- 对话
- 智能体
- 任务
- 我的

已在底部 Tab 中固定，不再漂移。

### 2. 首页收窄为真正驾驶舱
首页现在重点展示：
- AI 产出流
- 调度链
- 需确认项
- 记忆 / 知识 / 项目 / 附件四个信息层入口
- 当前焦点与最短路径闭环说明

已移除首页对“开发者自嗨信息”的依赖，不再把内部工程噪音放在首屏主视觉。

### 3. 对话链路更像真实产品
对话页已明确表达：
- 上下文不做产品层硬限制
- 附件上传是统一入口
- 前端不写死大小限制
- 记忆库 / 知识库 / 附件库 / 调度链可从对话页直接跳转

### 4. 信息层入口已经串起来
以下模块现在可从首页或“我的”页进入：
- 记忆库
- 知识库
- 项目库
- 附件库
- 调度链
- 需确认项

### 5. P1 的产品叙事已经收敛
当前移动端对用户传达的是：
- 这是 AI 大脑驾驶舱
- 不是普通 IM
- 不是飞书外壳
- 不是前端 demo 拼贴
- 核心价值是“接指令、看状态、追调度、管记忆、用知识、收附件”

## 还没完成

### 1. 真实 API 还未接入
当前仍以 mock 数据为主，下一步要优先替换：
- Agent 列表
- Task 列表
- Chat
- Upload
- Confirmations

### 2. 附件上传还只是产品壳
目前已表达产品策略，但还没有真正接入：
- 分片上传
- 断点续传
- 后台处理队列
- 结果回流状态

### 3. 记忆库 / 知识库还未接真实存储
当前页面结构已在，但需要后续接 OpenClaw 真实接口与检索链。

### 4. 上架链路还未真正开始
TestFlight / App Store 的工程链路仍待补：
- 图标、启动图、隐私描述
- 构建配置
- 发布流程文档

## 下一步建议

1. 先接真实 API，优先 Chat / Task / Agent / Upload。
2. 再把附件上传做成真实异步状态流。
3. 然后补记忆库 / 知识库真实检索。
4. 最后再转 TestFlight / App Store 上线链路。

## 设计边界

- React Native 是唯一主线。
- 不回退做 HTML 体验稿。
- 不在产品层做人为上下文截断。
- 不在前端写死附件大小限制。
- 首页只放用户价值，不放开发者自嗨信息。
