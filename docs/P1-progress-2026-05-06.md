# AIBrainIM P1 进展记录（2026-05-06）

## 本轮完成

### 1. 真实接入层往前推进了一步
本轮没有再堆新页面，而是把现有 React Native 主工程的"数据入口层"从纯 mock 推到了 **API 优先 + mock 回退** 的形态。

已补齐：
- `fetchAgents()`
- `fetchTasks()`
- `fetchConfirmations()`
- `resolveConfirmation()`
- `sendMessage()` 返回 `reply + taskId`
- `uploadFile()` 保持无前端硬限制策略

同时兼容两类返回结构：
- 直接数组
- `{ items: [...] }` / `{ data: [...] }`

这意味着后面接 OpenClaw / platform-server 真接口时，不需要再推翻页面，只需要逐步对齐字段。

### 2. 确认流不再只是静态展示
`AppContext` 现在刷新时会并行拉：
- Agent
- Task
- Confirmation

确认项的"确认 / 延后"操作也不再只是前端自娱自乐：
- 前端先本地更新，保证手感
- 同时异步尝试调用后端 `PATCH /confirmations/:id`

这一步把"需确认项"真正推进成了 P1 闭环的一部分。

### 3. 对话链路开始带调度单号语义
`ChatScreen` 本轮新增：
- 发送中状态
- 输入框锁定与发送按钮状态切换
- 后端返回 `taskId / dispatchId` 时，直接在对话结果里展示"调度单号"
- 保留"长上下文由后端处理，不在前端做人为截断"的产品边界

这一步很关键，因为它把"发一句话"从普通聊天，往"提交一张调度单"推进了一层。

### 4. 总览 / 任务 / 智能体页增加真实状态语义
本轮没有重做 UI，而是给现有页面补了更重要的信息：
- 当前是否正在同步
- 是否已经具备 API 接入能力
- 失败时是否回退 mock

这样首页不再只是漂亮卡片，而是开始表达"当前系统到底是真数据、回退数据、还是同步中"。

### 5. 对话调度单已经真正串到任务页和调度链页
本轮把"发出去一条消息"继续往前推进了一步。

已补齐：
- 任务页开始汇合显示两类任务：原始任务流 + 对话生成的调度单任务。
- 调度链页新增最新调度单摘要，直接显示 `taskId / dispatchId / status / session`。
- 调度链页新增最近调度记录列表，不再只看一条静态示意链。

这意味着移动端里"发指令 → 生成调度单 → 在任务和调度链里继续追踪"这条路径已经更像真实控制台，而不是演示稿。

### 6. 附件上传完整链路已打通
`uploadService.ts` 已实现完整架构（2026-05-06 23:xx 补齐）：
- **无前端大小限制**（不写死 maxSize）
- 小文件（<10MB）直传，大文件（≥10MB）分片（2MB/片）
- **断点续传**：失败只重试当前分片，已完成分片不重复上传
- 指数退避 + full jitter 重试（最多 5 次）
- **后台处理队列**（Promise 非阻塞，不卡 UI）
- 完整状态机：queued → uploading → processing → dispatched → done / error
- 失败可手动重试，从上次成功的分片继续
- Agent 自动分派（image/video→黑金，document→智联，archive→寻龙）

`UploadScreen`、`FileLibraryScreen`、`ChatScreen` 均接入同一队列。

### 7. CI/CD 链路已打通 TestFlight
- GitHub Actions: push main → TypeScript + iOS Simulator Build
- Tag v*.*.* → 自动 Archive + Export + TestFlight Upload
- Fastfile: `fastlane sim`（本地 Simulator）/ `fastlane tf`（TestFlight）
- ExportOptions.plist 已修正为 `app-store-connect` 方法
- AppIcon.appiconset 已创建（需替换 1024x1024 PNG）
